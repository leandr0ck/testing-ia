# AGENTS.md

Quick guide for agents working on this project.

## Project Overview

- **Name:** `hono-api`
- **Type:** Minimal HTTP API with **Hono** + **Bun** + **SQLite** + **Drizzle ORM**
- **Entry point:** `src/index.ts`
- **Local database:** `sqlite.db`

## Actual Stack

- **Runtime:** Bun
- **HTTP Framework:** Hono
- **ORM:** Drizzle ORM
- **DB:** SQLite (`bun:sqlite`)
- **Migration / schema tooling:** drizzle-kit + manual migration script
- **Tests:** `bun test`

## Key Commands

```bash
bun install
bun run dev

bun run db:migrate

bun run db:push

```

## 5. How the App Works

### `src/index.ts`

The app:

- responds to `GET /` with a welcome payload
- mounts routes at:
  - `/users`
  - `/tenants`
- applies a simple guardrail only to `/tenants/*`:
  - requires `x-workspace` header
  - if missing, responds `401 { error: 'Unauthorized' }`

### Available Routes

#### Users

- `GET /users`
- `GET /users/:id`
- `GET /users/:id/tenant`
- `GET /users/:id/profile`
- `GET /users/:id/full`
- `GET /users/profiles/all`

#### Tenants

- `GET /tenants`
- `GET /tenants/:id`
- `GET /tenants/:id/owner`
- `GET /tenants/:id/full`

> Important: any request to `/tenants/*` needs `x-workspace`.


## Conventions for Changes

### Do this

- Keep changes **small and surgical**.
- Read first:
  - `src/index.ts`
  - the affected route in `src/routes/*`
  - the associated query in `src/db/queries/*`
  - relevant tests in `test/*`
- If you change JSON contracts, update or add tests.
- If you change schema or data shape, check seed and migrations.

### Avoid this

- Don't migrate the app from Hono/Bun to another stack.
- Don't replace Drizzle with raw access unless it's a narrow fix.
- Don't use `src/data/*` for "quick fixes" on endpoints that should now come from SQLite.
- Don't do large refactors without need.

## Recommended Strategy by Change Type

### If the bug is in an endpoint

1. Reproduce with a test or manual request.
2. Review the route in `src/routes/...`.
3. Review the query in `src/db/queries/...`.
4. Confirm the serializable shape of the response.
5. Run `bun test`.

### If adding a new endpoint

- Define route in `src/routes/...`
- Implement query/helper in `src/db/queries/...`
- Add test in `test/...`
- If it depends on `/tenants/*`, remember `x-workspace`

## Expected JSON Responses

Pattern currently used:

- success: `{ data: ... }`
- error: `{ error: '...' }`

Root:

```json
{
  "message": "Hello Hono!",
  "endpoints": {
    "users": "/users",
    "tenants": "/tenants"
  }
}
```
